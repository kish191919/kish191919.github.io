1. Set_up

1) Go to https://azure.microsoft.com/en-us/free/students/ and click on the Activate now button

2) click on Create a resource.

3) Do a search for Hortonworks and click on the option for Hortonworks Data Platform (HDP) Sandbox

4) Click on the Create button. From here we just need to customize our sandbox.

5) On the Basics tab, click on Create new for the resource group. You can call your resource anything you want, e.g. DS730. The Virtual machine name can be called Hortonworks. Under Instance Details, look at the Size option. We are only allowed 4 cores with our student account. Therefore, click on the Change size link and choose the Standard B4ms option. Change the Administrator Account Authentication type to be a password. It is less secure than an SSH public key. Create any username/password you want.

6) Click create button

7) Click “Go to resource”

8) Click “Networking”

9) Click on the Add inbound port rule button. Add the following port using these rule:

Destination port ranges - Set it to be 8080. It might default at that port already. If that is the case, then leave it.

Name - Not critical but change it to Ambari so you know what the port is.

The rest of the options can be left at their defaults. Feel free to change the Source if you are super concerned about security.

10) Add the following Port/Name inbound port rules as well: (4200/ssh, 9995/zeppelin, 2222/hortonworksfs) You may have to change the priority to be a different number for each rule. Change it to any number that is unique.

11) Your Hortonworks Sandbox is setup and is now ready to use.

A great guide to learning about your Sandbox is here:

https://hortonworks.com/tutorial/learning-the-ropes-of-the-hortonworks-sandbox

However, everything you need to know for this course will be shown in this and future activities.

12) Go to “All resources”

13) Find your Public IP address on the right hand side

Note that my IP address is 40.88.11.48 and I will use that IP in the steps below.

14) Go to http://40.88.11.48:8080 and enter in maria_dev as the username and password. You should see something like this:

This step is important for all future activities and projects whenever you start up your virtual machine. You must check this page before beginning any activity/project to ensure the system is ready to use. As long as there are 0 alerts at the top, you are ready to start working. If you see any alerts, you must wait for the alerts to clear before starting. Even though you may be able to load up the 8080 port and the 4200 port, it does not mean the machine is completely started and ready to use. You must wait for all alerts to clear before beginning.

15) If you forget to do these steps, your virtual machine will still be running and you will run out of credits
