  • Data Lake :
    1) Data Structure : Raw
    2) Purpose of Data : Not yet Determined
    3) Users : Data Scientists
    4) Accessibility : High / Quck to update

  • Data Warehouse :
    1) Data Structure : Processed
    2) Purpose of Data : In Use
    3) Users : Business Professionals
    4) Accessibility : Complicated / Costly

  • Presto (SQL)
    1) Presto is a high performance, distributed SQL query engine for big data.
    2) One can even query data from multiple data sources within a single query

  • Severless : Lambda (User가 늘더라도 workload가 컨트롤 됨) , Athena
    1) 시스템 확장성, 가용성 측면에서 매우 뛰어남.
    2) 서버 환경을 구축할 때 비용이 드는데, 사용 고객이 가변적인 경우 자원 낭비는 필연적이며 이러한 제약을 극복하기 위해, serverless라는 개념이 등장하였다
    3) Request가 오면 그에 맞게 리소스를 선형적으로 늘려나감(병철 처리 기반)

  • Athena :presto 기반의 AWS 빅데이처 처리 리소스(플랫폼)
    1) partion을 기반으로 s3에 저장된 data를 빠르게 가져와서 ad-hoc분석을 쉽게 해 준다.
    2) 복수의 데이터 소스들로 부터 데이터를 합치기에 용이하다.
    3) nested 형식의 데이터를 파싱하는 강력한 쿼리 메소드를 지원한다.
    4) partition별로 저장된 parquet 포멧의 데이터를 다룰 수 있음.

  • Create table at Athena

  • Use partition

  • 반드시 Partition column을 사용하여 데이터를 query함. otherwise, all data will be caculated.

  • AWS EMR
    1) Amazon EMR은 AWS에서 Apache Hadoop 및 Apache Spark와 같은 빅데이터 프레임워크 실행을 간소화하는 관리형 클러스터 플랫폼입니다.
    2) 아마존 S3는 HDFS API를 만족시켜서 HDFS 대신 파일시스템으로 사용될 수 있고, 아마존 EMR은 EC2인스턴스 클러스터를 구성하여 하둡 프레임워크로서 동작하여 분산처리가 가능하게 합니다.
