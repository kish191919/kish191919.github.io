Used-Car-Price-Prediction-in-Virginia

This project demonstrates an end-to-end machine learning pipeline to predict used car prices in Virginia using structured tabular data, regression models, and advanced feature engineering.

📓 Jupyter Notebook View
👉 Click here to view the full notebook on GitHub

📊 Project Overview

Using a real-world dataset containing over 46,000 used car listings, the system is built on an AWS EC2 instance with the following key components:

🔹 Data Collection & Ingestion

  • Data was crawled and stored in an AWS-hosted MySQL database
  • Loaded via Python and served using Flask

🔹 Feature Engineering & Preprocessing

  • Structured features: mileage, model, year, transmission, fuel type, etc.
  • Handled missing data, transformed categorical variables, and removed outliers using IQR
  • Applied log transformation and normalization for skewed features

🔹 Exploratory Data Analysis (EDA)

  • Visualized distributions, outliers, and correlations using Seaborn & Matplotlib
  • Performed correlation analysis and VIF calculation to reduce redundancy
  • Found that year and mileage are most correlated with price

🧠 Model Development & Evaluation

A variety of regression models were trained and evaluated:

  • Linear Regression
  • Decision Tree
  • Random Forest
  • Support Vector Regression (SVR)
  • XGBoost Regressor

📈 Evaluation Metrics

  • Mean Absolute Error (MAE)
  • Root Mean Squared Error (RMSE)
  • R² Score (coefficient of determination)

✅ XGBoost Regressor outperformed all models with:

  • R² ≈ 0.84
  • RMSE ≈ 6950

🧪 Key Learnings

  • Normalization and log transformation improved model performance
  • GridSearchCV significantly optimized tree-based model results
  • Tree models handle high-dimensional categorical variables better

📌 Results Summary

Model R² Score RMSE
Linear Regression 0.58 14085
Decision Tree 0.71 9800
Random Forest 0.84 7021
Support Vector Reg. 0.60 10000
XGBoost 0.84 6950

🔍 Feature Importance & Real-World Prediction

  • Used XGBoost feature importance to select top features (Year, Mileage, Drivetrain, Engine, etc.)
  • Removed low-impact features like color and model name from final model
  • Conducted real-world test: Predicted price for 2016 Honda Odyssey: $18,738 → matched expectations from dataset comparison

🚀 Deployment

  • Final XGBoost model was serialized using Pickle
  • Served via a lightweight Flask API for real-time inference
  • A simplified version using only 4 features: Year, Mileage, Brand, Model was also created for web app integration

🧩 Technologies Used

Layer Tech
Data Source AWS MySQL
Backend Python, Flask
Modeling scikit-learn, XGBoost
Deployment Pickle, REST API
Platform AWS EC2 (Ubuntu 22.04)

💡 Takeaways

  • Log transformation significantly improves skewed regression targets
  • Tree-based models like RandomForest and XGBoost are robust with mixed data types
  • Feature importance enables smart dimensionality reduction
  • Even with limited features, real-time price estimation is possible and accurate

🌐 Web App Ready

  • A lightweight model (trained with only 4 fields) powers the prediction backend
  • Efficient enough for integration into Flask or Streamlit apps for public use
