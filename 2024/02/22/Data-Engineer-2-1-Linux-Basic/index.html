Data_Engineer_2_1_Linux_Basic

2024-02-22
2024-03-03
linux_basic

  1. man

    • 원하는 명령어의 설명 (Manual) 을 볼수 있음
    • ex) man find : find 명령어 볼수 있음
    • 기본 명령어가 아닌경우 -help, –help의 파라미터로 설명을 제공하는 명령어가 있음 -> 화면에 출력되기때문에 지저분할수 있음
    • J - 다운, K - 업 , Q - 종료, H - help

  2. cd

    • cd .. : 상위 디렉토리로 이동
    • cd etc/ 다음에 탭을 2번 연달아 눌러주면 모든 서브 디렉토리가 표시
    • cd - : 이전에 갔던 디렉토리로 되돌아감

  3. pwd

    • 절대 경로를 표시

  4. ls

    • ls -il : 파일 또는 디렉토리의 inode number를 표시
    • inode란 리눅스에서 파일 또는 디렉토리의 고유 식별 번호로서 파일 이름이 바뀌어도 유지
    • 파일 숨기기 (파일이름 앞에 .을 붙이면 됨) : mv hello.txt .hello.txt

  5. df

    • df -h : 디스크 공간 정보 , 인간이 읽을 수 있음 (h)

  6. du

    • du -sh : 해당경로 하위의 총합 (s), 인간이 읽을 수 있음 (h)
    • du -h -d 2 : 해당 depth 만큼 보여줌

  7. chmod

    • chmod +x test.txt : test 파일의 오너, 그룹, other 모두 x 가 추가
    • 실행 : ./test (앞에 ./ 를 붙여서 실행)

    chmod u+x test.txt : test 파일의 오너만 x 가 추가
    chmod g+w,o-r test.txt : test 파일의 그룹 w 가 추가, other에게는 r를 제거.
    chmod 760 test.txt

  8. head

    • head -n 3 : 첫 3줄만 출력

  9. tail

    • tail -f

  10. comm

    • comm -12 line1.txt line2.txt : line1 과 line2 공통 라인 출력
    • comm -3 line1.txt line2.txt : line1 과 line2 다른 라인 출력

  11. cmp

    • 두 파일을 바이트 단위로 비교한 결과를 출력

  12. diff

    • 두 파일의 라인별 차이점을 보여줌

      symbol 차이
      a : add
      c : change
      d : delete

  13. less

    • 터미널 세션에 프린트하지 않고 파일내용 볼수있음
    • 보는 부분만 열기때문에, 큰 용량의 파일을 볼때는 vi보다 빠름
    • 양방향 탐색이 가능하기때문에 more보다 편리

    j : down 5j 다섯줄 아래 space : 한 화면씩 아래
    k : up 5k 다섯줄 위 b : 한 화면씩 위
    shift + g : 맨아래
    h : help

  14. ln

    • 심볼릭 링크 : 절대경로 또는 상대경로 형태로 된 다른 파일이나 디렉토리에 대한 참조를 포함한 특별한 종류의 파일

    • in -s $원래파일위치 $목표파일위치

    • 원래 파일을 삭제하면 심볼릭 링크로 만든 파일에서 에러가 발생

    • 파일에 심볼릭 링크 설정

    • 폴더에 심볼릭 링크 설정

  15. alias

    • 다른 문자열에 대채하는 단어 지정
    • 주로 복잡한 명령어의 약어로 사용
    • 환경변수와 다름

  16. cp

    • cp -r 디렉토리 타겟디렉토리 :

  17. find

    • find . -name 파일이름 : 파일을 찾음
    • find . -name ‘정규표현식’

  18. which

    • $PATH에 등록된 경로중에 주어진 이름의 실행 파일의 위치 보여줌
    • 사용하는 명령어의 위치 찾음

  19. grep

    • grep -rl “[0-9]0{3}“ . : 현재 디렉토리에서 하위 디렉토리까지 검색하여 맞는 조건의 파일이름만 표기
    • grep -n “[0-9]0{1}“ find.txt : 찾는 문자열의 줄번호 표기

  20. sed

    • 텍스트를 필터링하거나 변환하는 스트림 에디터
    • sed s/old_word/new_word/g target_file : 타겟파일에서 old word를 new word 로 교체
    • vi에서도 %s/old_word/new_word/g 를 이용하여 단어를 교체

  21. uname

    • 이름, 버전, 기타 시스템 정보확인
    • -a : 모든정보 표기

  22. ps

    • 현재 실행중인 프로세스 확인
    • PID : 프로세스 아이디
    • PPID : 부모 프로세스 아이디
    • STIME : 프로세스 시작 시간
    • TTY : 프로세스가 실행된 터미널의 종류와 번호. ?는 없다는 의미.
    • TIME : 프로세스에 의해 사용된 CPU 시간
    • CMD : 실행된 프로세스의 이름 혹은 실행된 명령

    -e : 현재 사용자뿐만아니라 다른 사용자들이 구동시킨 모든 프로세스 보여줌
    -f : 보다 상세한 정보
    -l : f보다 더 상세산 정보

    • 참고문헌 : https://tigris-data-science.tistory.com/entry/Linux-ps-%EB%AA%85%EB%A0%B9%EC%96%B4
    • ps aux : 프로세스 상태들을 cpu와 메모리 사용량을 표시

  23. kill

    • process 에 signal을 전달

    kill -9 $pid : 프로세스를 강제 종료
    kill -15 $pid : 종류하겠다는 signal을 보냄. 자동화된 프로세스에서 주로 사용됨 (추가 공부 필요)
    kill -2 $pid : interrupt를 의미하는 signal. ctrl+c를 누르면 이 signal이 자동으로 전달됨

  24. ss (netstat for Mac)

    • socket status 의 약어
    • 네트워크와 소켓의 사용을 쉽게 확인. 주로 열려있는 포트를 확인할때 사용
    • 내용이 긴 경우 pipe() 해서 less 또는 grep을 사용

    ss : 연결이 맺어진 (status = ESTAB) 소켓 확인
    ss -a : 열려있는 모든 소켓
    ss -l : listening 중인 소켓만 확인
    ss -t : TCU 소켓만 확인
    ss -u : UDP 소켓만 확인
    ss -s : 소켓 상태 통계를 나타냄

  25. echo

    • 터미널 콘솔에 텍스트를 출력
    • echo $(명령어) 명령어의 결과를 출력
    • 파이프와 함께 여러 용도로 사용

  26. history

    • 지나간 명령어 기록을 확인
    • history에 확인된 명령어의 숫자를 !number로 다시 사용
    • 로그아웃 되거나 세션이 종료될때는 다시 history가 안보임 => 그러나 .zsh_history 에 저장이되어 세션이 종료되어도 명령어들을 볼수 있음

  27. redirection

    • 출력결과를 파일로 저장

  28. stdout, stderror

    • Process 표준 입출력 출력

  29. 환경변수

    • 변수에 자주 사용하는 경로를 저장함

    • 기존에 있는 변수를 이용한 새로운 변수를 저장 : PATH=$PATH:newpath

    • 새 터미널 열때도 계속 동작하게 하려면 : vi ~/.bash_profile 수정.
      단, bash shell 로 접속했을때만 동작. sh 또는 zsh로 접속하면 동작하지 않음

    • 환경변수를 모든 유저에게만 영구적으로 적용하고 싶을경우

    • 실행명령어 만들기

    • 보통 만들어진 명령어들은 /usr/local/bin 에 저장
