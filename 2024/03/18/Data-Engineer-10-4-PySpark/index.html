1. Python 집계 연산

[과제1] 2008년 비행기록 데이터에서 월(month) 별 비행회수, 도착지연시간 총합/평균/최소/최대 값

[과제2] 과거 모든(1997년 ~ 2008년) 비행기록 데이터에서 연도(year)/월(month) 별 비행회수, 도착지연시간 총합/평균/최소/최대 값

(1) Python 설치

(2) Python 집계 연산 - 과제1 (2008 , DataFrame API , pandas) -> 00:00:05…. -> cpu 1개만 열심히 사용

(2) Python SQL 집계 연산 - 과제1 (2008 , SQL문 , pandasql) -> Killed….

(3) Python SQL 집계 연산 (데이터 양을 줄임 1000000 ) - 과제1 (2008 , SQL문 , pandasql) -> 00:00:33

(4) Python 집계 연산 (모든 데이터) - 과제2 (all , DataFrame API , pandas) -> Killed

(5) PySpark - 집계 연산 (on local[*]) - 과제1 (2008 , DataFrame ) -> cpu 2개 모두 열심히 사용 00:00:05

(6) PySpark - 집계 연산 (on local[*]) - 과제1 (2008 , SQL문 ) -> cpu 2개 모두 열심히 사용 00:00:06

  • 그냥 SQL 문으로 사용할때는 33초 걸렸음

  • 판다스에서는 집계하고자 하는 것을 모두 메모리에 올리기때문에 메모리가 부족

  • 파이스파크는 분석하고자 하는 데이터를 전체 메모리에 올리지 않음. 필요한 것만 메모리에 올려서 처리함 => 메모리를 많이 필요하지 않음

(7) PySpark - 집계 연산 (on local[*]) - 과제2 (all , DataFrame API )-> cpu 2개 모두 열심히 사용 00:02:36

(8) PySpark - 집계 연산 (on local[*]) - 과제2 (all , SQL문 )-> cpu 2개 모두 열심히 사용 00:02:36

(9) PySpark 집계 연산 (cluster with yarn) -과제1 (2008 , DataFrame API) 00:00:03 -> 각 executor의 cpu 2개 모두 열심히 사용

  • 모든 워커서버에서 아래 있는 파일들을 카피해감
  • 각 워커서버의 executor들이 2개씩 작동함

(10) PySpark 집계 연산 (cluster with yarn) - 과제1 (2008, SQL문 ) -> 00:00:03 -> 각 executor의 cpu 2개 모두 열심히 사용

(11) PySpark 집계 연산 (cluster with yarn) - 과제2 (all , DataFrame API) -> 00:00:55 -> 각 executor의 cpu 2개 모두 열심히 사용

(12) PySpark 집계 연산 (cluster with yarn) - 과제2 (all , SQL문) -> 00:00:54 -> 각 executor의 cpu 2개 모두 열심히 사용

(13) 종료되면 HDFS에 있던 파일이 삭제됨
