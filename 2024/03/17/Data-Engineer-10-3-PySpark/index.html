Standalone

1. Spark Standalone mode 설정

  • Spark 다운로드 완료
  • Java 다운로드 완료
  • spark-env.sh 에 JAVA_HOME 설정 완료
  • Copy ‘jdk8’ spark-worker-01 ~ 03 완료

(1) workers 설정 - spark-master-01

(2) copy ‘spark3’ to spark-worker-01 ~ 03

2. Standalone Cluster Manager 시작

(1) Spark standalone cluster manager 시작 - spark-master-01

(2) spark-env.sh 설정 (host_name, port, cores, memory 설정….) - spark-master-01

(3) copy ‘spark-env.sh’ to spark-worker-01 ~ 03

(4) spark standalone 재기동 + 웹UI 확인

(5) spark standalone 웹UI (재)확인 - 방화벽 변경, Spark Master HTTP, 8080 -> 8180

  • http://spark-master-01:8180
  • host_name, port, cores, memory 설정 반영 여부 확인….

(6) Worker Id 링크를 클릭하여 Worker 상세화면 확인 - 방화벽 오픈, Spark Worker HTTP, 8181

(7) Standalone ‘Master’만 시작 / 종료 - spark-master-01

(8) Standalone ‘Worker’를 추가 - spark-worker-01 ~ 03

  • spark-worker01를 추가할 경우
  • 이렇게 하나씩 띄우고 삭제하는 이유는 워커서버가 고장났을 경우, 하나씩 따로 내리고 다시 올릴 수 있음
  • Private IP address에 Spark-worker-01:7178로 나타남 (–host ${HOSTNAME} 를 줬기때문임)
  • 7178 포트 설정은 명령어에 없는 설정이기때문에 /data/spark3/conf/spark-env.sh 에서 갖고옴
  • worker를 정지할 경우, 마스터 UI에서 그 Worker의 상태는 ‘DEAD’로 표시됨

3-1. Standalone Cluster Manager 테스트 1

(1) 모든 워커 시작

(2) spark-shell 실행

(3) spark-shell 확인

(4) Spark App 웹UI 확인….

  • http://spark-master-01:4040
  • Executors > executor 3개 할당된 것 확인 (-> Cores 8, ALL Cores) => 클러스터의 모든 core 할당 받음

(5) Standalone 웹UI 확인….

  • http://spark-master-01:8180
  • Running Applications > (Name / Cores / Memory per Executor 확인) => 8 cores 모두 사용되고 메모리는 1기가만 부여됨
  • Running Applications > Application ID > Executor Summary > (ExecutorID / Cores / Memory 확인….) (-> Cores 8, ALL Cores ????) (-> Memory 1024, only 1024.0 MiB ????)
  • Running Applications > Application ID > Executor Summary > Worker > (Master URL / Cores / Memory 확인….)
  • Running Applications > Application ID > Executor Summary > Worker > Running Executors > (Cores / Memory / Job Details 확인….)

3-2. Standalone Cluster Manager 테스트 2

(1) 기존 spark-shell 닫기

(2) spark-shell 실행

(3) Standalone 웹UI(에서도) (재)확인…. (-> executor 6개, cores 2개 Good!!!!)

  • http://spark-master-01:8180
  • Running Applications > (Name / Cores / Memory per Executor 확인….)
  • Running Applications > Application ID > Executor Summary > (ExecutorID / Cores / Memory 확인….) (-> Cores 2, OK!) (-> Memory 2048, OK!)
  • Running Applications > Application ID > Executor Summary > Worker > (Master URL / Cores / Memory 확인….)
  • Running Applications > Application ID > Executor Summary > Worker > Running Executors > (Cores / Memory / Job Details 확인….)

(4) Spark App 웹UI 확인…. (-> executor 6개, cores 2개 Good!!!!)

  • http://spark-master-01:4040
  • Executors > executor 6개 할당된 것 확인…. executor 당 cores 2개 할당된 것 확인….

(5) driver 프로세스 확인 - spark-master-01

(6) executor(6개) 프로세스 확인 - spark-worker-01 ~ 03

4. Standalone Cluster Manager 디폴트 설정 -> 모든 리소스 쓰지 않도록 함

(1) spark-env.sh 설정 - spark-master-01

(2) copy ‘spark-env.sh’ to spark-worker-01 ~ 03….

(3) spark standalone 재기동…. + 웹UI 확인….

(4) spark-shell 실행…. (w/ –master spark://host:port) (w/o memory,cores,total 옵션)

(5) spark-shell (동시에) 실행….#2 (w/ –master spark://host:port) (w/ memory,cores 옵션) (w/o total 옵션)

(6) Standalone 웹UI 확인…. (-> NO ALL Cores, Good!!!!) (-> –total-executor-cores(=spark.cores.max) 옵션을 깜빡해도 이제 어느정도 안전하다 ^^)

  • http://spark-master-01:8180
  • Running Applications > Application ID > Executor Summary > (ExecutorID / Cores / Memory 확인….)
  • Total Core는 디폴트 값에 의해서 5개 안에서 해결해야 함으로 Executor는 2개만 할당받음

5. RDD나 셔플 정보를 로컬에 저장할 장소 설정 - 02-32

(1) spark-env.sh 설정….#2 (SPARK_LOCAL_DIRS=/data/spark3/local 설정….) - spark-master-01

  • 디폴트 “/tmp” 에 셔플과 케쉬데이터가 있을경우, 서버가 재기동 될때 모두 사라짐
  • 확인을 위해서 위치를 바꿈

(2) copy ‘spark-env.sh’ to spark-worker-01 ~ 03….

(3) spark standalone 재기동…. + 웹UI 확인….

(4) spark-shell 실행…. (w/ –master spark://host:port) (w/o memory,cores,total 옵션)

(5) Word Count 코드 작성…. (Shuffle Data Write/Read….)

(6) LOCAL_DIRS 확인 (/data/spark3/local) - spark-master-01 & spark-worker-01 ~ 03

(7) RDD Cache (on Disk) (Spark App 웹UI > Storage > RDD Name > Block Name 확인….)
